# CTX-001: Context Pack CRUD and Mounting

## Goal

Implement durable context packs and mount records so shared project context can be attached to running agents with versioned snapshot semantics.

## Scope

- create and list context packs
- persist context items
- mount records per agent
- token budget metadata
- audit logging for context mounts

## Deliverables

- DB-backed context pack repository
- REST handlers backed by persistence
- mount endpoint that records versioned attachments
- tests for immutability and remount behavior

## Acceptance checks

- context packs are persisted with immutable version numbers
- mounts are queryable by agent and workspace
- remounting a new pack version does not mutate historical records
- audit trail captures mount timestamp and requested token budget

## Dependencies

- depends on `DB-001`
- should align with `ORCH-001` event emission behavior
