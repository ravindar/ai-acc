# RPL-001: Session Replay API and Timeline

## Goal

Expose historical event streams and build a replay-friendly timeline so finished sessions can be inspected, audited, and shared.

## Scope

- paginated event retrieval
- timeline view model
- replay cursor semantics
- artifact listing by session
- context mount visibility in replay

## Deliverables

- replay API endpoints
- desktop timeline panel
- event cursor contract
- tests for replay ordering and completeness

## Acceptance checks

- finished sessions can be replayed from persisted events
- ordering is stable and complete across page boundaries
- context mounts and tool calls appear in the replay timeline
- artifacts associated with the session are discoverable

## Dependencies

- depends on `EVT-001`
- depends on `DB-001`
