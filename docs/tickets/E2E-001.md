# E2E-001: Mixed-Provider Smoke Harness

## Goal

Prove the v1 slice works end to end by launching a representative multi-agent workspace and verifying the primary operator flows under load.

## Scope

- seeded workspace scenario
- 10-agent smoke run using deterministic adapters
- grid rendering verification
- status transition verification
- replay and usage summary verification

## Deliverables

- automated smoke script or suite
- test fixtures for workspace bootstrap
- pass/fail report format
- developer instructions for running the smoke path locally

## Acceptance checks

- 10 sessions can be created and observed from one workspace
- running, idle, waiting, and error states all surface correctly
- usage and replay data appear for completed sessions
- the desktop shell remains usable throughout the scenario

## Dependencies

- should be tackled after `ORCH-001`, `ADP-001`, `TEL-001`, and `UI-001`
