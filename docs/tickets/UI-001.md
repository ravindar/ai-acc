# UI-001: Desktop Grid and Focus Shell

## Goal

Turn the desktop mock into a data-driven shell with workspace grid rendering, focus mode, filters, and side panels connected to the local control plane.

## Scope

- workspace selection
- grid tile layout state
- focus mode
- status and filter controls
- usage and context side panels

## Deliverables

- websocket client and query layer
- agent tile components bound to live data
- monitor-aware layout state model
- focus view route and interactions

## Acceptance checks

- live sessions render in the grid without manual refresh
- idle and error filters behave correctly
- focus mode preserves grid state when exiting
- the shell remains responsive with at least 10 active mock sessions

## Dependencies

- depends on `EVT-001`
- strongly benefits from `ADP-001`
