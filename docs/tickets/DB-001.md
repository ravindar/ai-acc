# DB-001: Baseline Persistence

## Goal

Replace the temporary in-memory store with real Postgres-backed persistence and establish a repeatable migration workflow.

## Scope

- choose migration runner
- wire database connection lifecycle
- apply the baseline schema from `0001_initial.sql`
- implement repository methods for workspaces, agents, and context packs

## Deliverables

- migration execution path for local development
- typed DB access layer
- persistence-backed implementations for current REST handlers
- local developer setup notes for schema bootstrap

## Acceptance checks

- app can connect to Postgres using `DATABASE_URL`
- baseline migration applies cleanly on an empty database
- workspace and agent creation survive process restart
- no existing route depends on the in-memory store after completion
