# EVT-001: Normalized Event Pipeline

## Goal

Define and enforce the single normalized event contract used across adapters, persistence, replay, and the desktop client.

## Scope

- final event envelope schema
- validation and serialization helpers
- sequence assignment rules
- persistence model for event storage
- websocket publish contract

## Deliverables

- finalized event schema package
- serializer and validator utilities
- event insertion path in the control plane
- tests covering supported event types and invalid payloads

## Acceptance checks

- all emitted events validate before persistence
- sequence numbers are monotonic per agent
- invalid payloads fail with actionable errors
- websocket messages use the same schema as stored events

## Dependencies

- can start after `CP-001`
- should be completed before `RPL-001`
