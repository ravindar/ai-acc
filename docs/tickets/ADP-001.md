# ADP-001: Deterministic Mock Adapter

## Goal

Create a provider-neutral mock adapter that emits scripted events so the orchestrator, telemetry, and UI can be built without depending on live LLM runtimes.

## Scope

- adapter implementation using `@acc/adapter-sdk`
- deterministic event script format
- controllable timing for heartbeat, output, and failure events
- fixture scenarios for success, idle, waiting input, and error flows

## Deliverables

- mock adapter package or module
- reusable test fixtures
- event playback support for integration tests
- sample scenario wired into the control plane

## Acceptance checks

- same script produces the same event sequence across runs
- orchestrator can run a full session using the mock adapter
- UI can display active, idle, waiting, and error states from mock events
- no live provider key is required for core local development

## Dependencies

- depends on `EVT-001`
- should land before live Codex or Claude adapter work
